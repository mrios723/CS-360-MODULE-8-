1. Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?
The goal of my app was to create a simple sensor-based Android application that displayed real-time data from a device sensor. The app was designed to give users an easy way to view sensor information in a clear and understandable format while demonstrating how Android applications interact with built-in hardware.

2. What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?
The app included a single main screen with a title, a sensor value display, and status information. I kept the interface clean and uncluttered so users could quickly understand the information without distractions. Using readable text, logical placement of elements, and a simple layout made the app easy to navigate and use.

3. How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?
I broke the project into smaller tasks, such as creating the user interface first, then connecting the sensor, and finally testing each feature. I focused on solving one problem at a time and tested frequently as I coded. This approach helped reduce errors and can be used in future projects to make development more organized and efficient.

4. How did you test to ensure your code was functional? Why is this process important, and what did it reveal?
I tested the app in the Android Emulator and verified that the sensor values updated correctly without crashing. I also checked that the user interface displayed the information as expected. Testing is important because it helps identify bugs early and ensures the app performs reliably before it is released.

5. Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?
One challenge was learning how to connect the Android sensor to the user interface and handle updates in real time. I overcame this by reviewing Android documentation, troubleshooting errors, and making small changes until everything worked correctly. This experience improved my problem-solving skills and confidence with Android development.

6. In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?
I was most successful in implementing the sensor functionality and displaying the live sensor data within the app. This demonstrated my understanding of Android development, user interface design, and debugging while creating an application that met the assignment requirements.
